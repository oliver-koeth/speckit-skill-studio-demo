# Ralph Progress Log
## Codebase Patterns
- Keep the Angular app standalone-first: app bootstrap stays in `src/main.ts`, shared router setup stays in `src/app/app.routes.ts`, and route entry points should use `loadComponent`.
- Preserve the planned feature structure under `src/app/`: route pages belong in `features/<feature>/pages/`, while reusable cross-cutting code belongs under `core/` or `shared/`.
- This project is aligned to Playwright-only browser verification; do not add Karma/Jasmine scaffolding back unless the product requirements change.
- Shared app chrome should live in a standalone shell component mounted from `src/app/app.routes.ts`; keep feature pages as child routes so the header and navigation are defined once.
- Browser verification runs through `playwright.config.ts`, which starts `ng serve` on `127.0.0.1:4200`; store UI specs under `tests/` and ignore generated `test-results/` artifacts.
